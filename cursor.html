Cursor Coding Agent

坐在你 IDE 里的结对工程师
»»»

Cursor 适合观察编码 Agent 的工具偏好、文件编辑规则、上下文选择和开发者体验。它的 system prompt 重点不是人格陪伴，而是如何把模型变成能读 repo、改文件、跑命令的工程同事。 本页已接入 system_prompts_leaks 中对应真实 prompt 全文；下方时间线可打开原文、来源字段与逐项分析。

verbatim evidence
system_prompts_leaks / Cursor/cursor.md · identity_context
Cursor attaches open files, cursor position, recent files, edit history, and linter errors to the user turn.
Cursor 的“人格”来自 IDE 现场：它不是凭空聊天，而是带着当前仓库状态、编辑历史和错误信息进入任务。
system_prompts_leaks / Cursor/cursor.md · tone_tool_rules
Output text is for the user; tools are for completing tasks. Shell/code comments should not be used as communication.
这条把沟通和执行分开：给人看的话走回答，真正动作走工具，避免把终端当成聊天窗口。
system_prompts_leaks / Cursor/cursor.md · mcp_rules
Before calling MCP tools, Cursor must inspect tool descriptors and schemas.
Cursor 的工具调用纪律很工程化：先看 schema，再调用工具。这比“能调用 MCP”本身更关键。

二、人格成因拆解

why this personality
► 仓库优先
先读上下文再行动，是 Cursor 和普通聊天模型的分水岭。
► 最小修改
好的编码 Agent 应该少动无关文件，避免把任务变成重构秀。
► 工具链人格
搜索、编辑、终端、diff、测试共同构成它的工作风格。
► 开发者体验导向
回答不只要正确，还要让工程师知道改了什么、怎么验证。

三、人格画像 · 两种讲法

two audiences
面向大学生 · 科普版
坐你旁边的结对编程学长
你说 bug，他先打开 repo 看，不空口猜

Cursor 的好用不在于会聊天，而在于它知道代码要落在文件里、测试要跑起来。

它像一个守规矩的 pair programmer：少说废话，先把问题复现，再给补丁。

"坐在你 IDE 里的结对工程师：这是给非技术读者的一句话人格锚点。
面向 KA 高层 · 授课版
IDE 内嵌的工程执行层
适合研发提效，但必须绑定代码权限和审查流程

对企业来说，Cursor 价值在降低工程师上下文切换成本。

风险在于它能改真实代码，所以权限、审计、测试和回滚流程比人格更重要。

"企业价值不只看模型聪明不聪明，而是看它在权限、工具、审计和失败恢复里是否可控。
